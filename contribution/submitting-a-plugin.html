Submitting a Plugin

This document describes the steps for submitting a plugin to the MojoHaus project. These steps might not apply to everyone, but should be a general guideline.

  • Subscribe to the developer mailing list and ask if there is an interest in the plugin.
  • Point us the GitHub repository where your plugin code exist.
  • If there is an interest, make sure that you try to follow our development guidelines.

Once accepted, we will agree with you method of transferring your repository to MojoHaus GitHub organization.

After transferring all MojoHaus committers can continue to develop and maintain it, everybody will be able to create issues and provide a pull requests.

We will make first release under MojoHaus organization when we will be sure that it is ready to be used by all and meet our requirements.

Note

You will not automatically become a committer on the project. If you stick around and keep on maintaining the plugin we'll most likely grant you commit rights once we feel comfortable with you and your work.
